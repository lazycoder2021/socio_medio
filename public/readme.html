Apart from the usual auth and crud, this app also allows image/file uploading.

To upload user profile photo, you should first choose file and click submit, then click update profile button to update the image

You can follow/unfollow users. But the feed route (though fetches the followers posts correctly, I'm not able to store it in a variable and pass it to frontend, for now the data gets console logged on the server side) does not work.

Very basic counter value based like/dislike is available. I realized adding user id based likes/dislikes is also easy to add, but didn't include it for now.

You can add comments too

To upload images while creating posts, you should again first upload the image by choosing file and hitting submit button, then get the image url returned by the server, and use it in image url input box while providing other post details.

View your posts button does not work. I didn't code it for now.

I realized that I write naive code (when I compared it with others on Github) but I continued coding my way, because that's the only way you can explore and learn. Then use the efficient way possible.

The app runs on mongo atlas, so you can download github code and npm install dependencies and you'll be good to run the app. You don't even need .env file.

The best way to proceed further is to add a rudimentary chat/video calling facility (apart from fixing all the errors and making the screens more user friendly). For that I have to learn a bit about socket and webrtc.
